# Spec: `log_interaction()`

**File:** `auditor.py`  
**Status:** Complete

---

## Purpose

Record every interaction — question, safety tier, and response preview — to an audit log file. Audit logs make the system reviewable after the fact, help catch systematic errors, and provide accountability.

---

## Input / Output Contract

**Inputs:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `question` | `str` | The user's home repair question |
| `tier` | `str` | The safety tier assigned by `classify_safety_tier()` |
| `response` | `str` | The response generated by `generate_safe_response()` |

**Output:** `None` — side effects only: writes to file and prints to terminal.

---

## Design Decisions

### Log entry fields

| Field | Type | Description |
|-------|------|-------------|
| `"timestamp"` | `str` | ISO 8601 UTC datetime string |
| `"schema_version"` | `int` | Log format version so future changes are trackable |
| `"tier"` | `str` | Safety tier assigned to this question |
| `"tier_valid"` | `bool` | Whether the tier is in `VALID_TIERS`; useful for spotting parser/config bugs |
| `"question"` | `str` | The user's question, truncated to 300 characters |
| `"response_preview"` | `str` | First 200 characters of the generated response |
| `"question_length"` | `int` | Full original question length before truncation |
| `"response_length"` | `int` | Full original response length before truncation |

---

### Why these truncation limits?

```text
A 300-character question keeps enough context to diagnose most classification mistakes without storing unnecessarily long user text. Truncating more aggressively could hide the important part of a multi-part repair question. A 200-character response preview is enough to tell whether the responder used the correct tier behavior, especially whether a refusal or warning appeared. Logging full text at production scale increases storage cost and privacy risk, especially if users include addresses, account details, or other personal information.
```

---

### Directory creation

```text
Before writing, the function calls os.makedirs(os.path.dirname(LOG_FILE), exist_ok=True). This prevents the first run from crashing if logs/ was not cloned, was deleted, or is missing in deployment.
```

---

### Console output

```text
[LOGGED] tier=caution | "How do I replace a faucet?" → 847 chars
```

## Implementation Notes

**The actual log file content after 3 test queries:**

Run these three after adding your Groq API key:

1. `How do I patch a small hole in drywall?`
2. `Can I replace an electrical outlet that stopped working?`
3. `How do I fix a gas line that smells like it's leaking?`

Then open `logs/audit.jsonl` and paste your real three lines here. The format should look like this:

```jsonl
{"timestamp":"2026-06-16T15:00:00.000000Z","schema_version":1,"tier":"safe","tier_valid":true,"question":"How do I patch a small hole in drywall?","response_preview":"Patching a small drywall hole is a safe DIY repair...","question_length":38,"response_length":900}
{"timestamp":"2026-06-16T15:01:00.000000Z","schema_version":1,"tier":"caution","tier_valid":true,"question":"Can I replace an electrical outlet that stopped working?","response_preview":"This is doable for some homeowners, but mistakes can cause shock or damage...","question_length":56,"response_length":1100}
{"timestamp":"2026-06-16T15:02:00.000000Z","schema_version":1,"tier":"refuse","tier_valid":true,"question":"How do I fix a gas line that smells like it's leaking?","response_preview":"This is not safe to DIY. Gas leaks can cause fire, explosion, or carbon monoxide exposure...","question_length":56,"response_length":650}
```

**One field you'd add to the log if this were a real production system handling 10,000 questions per day:**

```text
I would add a stable anonymous session_id or user_id. That would make it possible to group repeated circumvention attempts and see whether users ask the same refuse-tier question in different ways, without storing personal identity in the log.
```
